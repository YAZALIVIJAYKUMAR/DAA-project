Algorithm Explanation

Algorithm Overview

The project uses a combination of algorithms to match user input ingredients with recipes and suggest suitable options. The key algorithms used are:

  • Greedy Matching: This algorithm quickly identifies recipes that contain the maximum number of user-selected ingredients. It forms the basis of filtering recipes that are relevant.
  • Gap Analysis: It analyzes the difference between recipe ingredients and user inputs to find out what ingredients are missing, enabling the project to inform users about missing items for each recipe.
  • Smart Substitutions: This module suggests alternative ingredients based on similarities and common substitutions for missing ingredients to make the recipe feasible.
  • Backtracking Search: This depth-first search algorithm examines various combinations and ingredient permutations to find optimized recipe suggestions that match user inputs as closely as possible.

Implementation Details

1. Data Preparation: All recipes and respective ingredients are prepared in JSON format.

2. Input Processing: User inputs a comma-separated list of ingredients.

3. Set Operations: Efficient set intersections and differences compute 'have' and 'missing' ingredients between user inputs and recipes.

4. Matching and Ranking: Recipes are ranked based on how many ingredients match the user's input using greedy approach.

5. Alternative Suggestions: For missing ingredients, the smart substitutions algorithm proposes possible ingredient replacements improving user experience.

6. Result Presentation: Matched recipes and their missing ingredients are shown with counts and ingredient pills on the UI for clarity.

Benefits

This combined algorithmic approach ensures a fast, intuitive, and helpful recipe suggestion system tailored to what ingredients the user actually has, reducing food waste and promoting efficient cooking.
